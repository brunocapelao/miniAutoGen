## Task 2: Write Quickstart Guide

**Problem:** No documentation tells a new user how to go from install to first run.

**Files:**
- Create: `docs/quickstart.md`
- Modify: `README.md` (add link to quickstart)

- [ ] **Step 1: Write quickstart**

Create `docs/quickstart.md`:

```markdown
# Quickstart

## Install

pip install miniautogen

## Create a Project

miniautogen init my-project
cd my-project

## Configure an Engine

Edit `miniautogen.yaml` to set your AI provider:

### OpenAI
engines:
  default:
    provider: openai
    model: gpt-4o

Set your API key: `export OPENAI_API_KEY=sk-...`

### Gemini CLI (free, no API key)
engines:
  default:
    provider: gemini-cli
    model: gemini-2.5-pro
    command: gemini

## Create Agents

Create YAML files in `agents/`:

agents/researcher.yaml:
  role: researcher
  goal: Research the given topic thoroughly

agents/writer.yaml:
  role: writer
  goal: Write clear documentation based on research

## Define a Flow

In `miniautogen.yaml`:
flows:
  research-and-write:
    mode: workflow
    participants: [researcher, writer]

## Run

miniautogen run research-and-write --input "Explain async Python"

## What Happens

1. PipelineRunner loads your config
2. Creates an AgentRuntime per agent (with memory, tools, hooks)
3. Executes the flow (workflow = sequential, deliberation = multi-round, loop = router-driven)
4. Events are emitted for observability (69 event types)
5. Results are returned

## Next Steps

- `miniautogen dash` — Open the TUI dashboard
- `miniautogen check` — Validate your project config
- Read `docs/pt/` for full architecture documentation
```
